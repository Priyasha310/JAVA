# Java Basics: Platform, JVM, JDK, JRE & Java Editions

## Java

Java is a **platform-independent**, **object-oriented** programming language designed to be simple, secure, and portable.

### Features of Java

* Platform Independent
* Object-Oriented
* Robust
* Secure
* Multithreaded
* High Performance (using JIT Compiler)
* Distributed
* Dynamic

### WORA (Write Once, Run Anywhere)

Java follows the principle of **WORA (Write Once, Run Anywhere)**.

A Java program is compiled once into **bytecode**, which can run on any operating system that has a compatible **JVM**.

---

# JVM (Java Virtual Machine)

### Definition

The **JVM** is responsible for executing Java bytecode by converting it into machine code that the operating system understands.

### Key Points

* Platform Dependent
* Executes Java Bytecode
* Contains the **JIT (Just-In-Time) Compiler**
* Performs Memory Management and Garbage Collection

### JIT (Just-In-Time) Compiler

The JIT Compiler converts frequently used **bytecode** into **native machine code** at runtime, improving application performance.

# JRE (Java Runtime Environment)

### Definition

JRE provides the environment required to **run** Java applications.

### Components

* JVM
* Core Java Libraries
* Supporting Runtime Files

### Key Point

* Platform Dependent
* Used only for **running** Java applications.

```text
JRE = JVM + Libraries + Runtime Files
```

---

# JDK (Java Development Kit)

### Definition

JDK provides everything needed to **develop, compile, and run** Java applications.

### Components

* JRE
* Java Compiler (`javac`)
* Debugger
* Documentation Tool (`javadoc`)
* Other Development Utilities

### Key Point

* Platform Dependent
* Used for Java Development.

```text
JDK = JRE + Development Tools
```

---

# Java Editions

## JSE (Java Standard Edition)

Used for developing **desktop, console, and core Java applications**.

Provides core Java APIs such as:

* `java.lang`
* `java.util`
* `java.io`
* `java.net`
* `java.sql`

Examples:

* Console Applications
* Desktop Applications
* Utility Programs

---

## JEE (Java Enterprise Edition) *(Now Jakarta EE)*

Built on top of JSE for developing **enterprise-level web applications**.

Provides APIs for:

* Servlets
* JSP
* RESTful Web Services
* Transaction Management
* Messaging (JMS)
* Enterprise Security

Examples:

* Banking Applications
* E-commerce Websites
* ERP Systems

---

## JME (Java Micro Edition)

Designed for **mobile and embedded devices**.

Provides lightweight APIs for:

* Embedded Systems
* IoT Devices
* Small Consumer Electronics

Examples:

* Smart TVs
* Embedded Controllers
* IoT Devices

# Understanding `public static void main(String[] args)`

```java
public static void main(String[] args) {
    System.out.println("Hello, World!");
}
```

This is the **entry point** of every Java application. The JVM starts program execution from the `main()` method.

---

## `public`

### Meaning

* **Access Modifier**
* Makes the `main()` method accessible from **anywhere**.

### Why is it needed?

The JVM must be able to access and invoke the `main()` method. If it is not `public`, the JVM cannot call it.

---

## `static`

### Meaning

* Belongs to the **class**, not to an object.

### Why is it needed?

The JVM calls `main()` **before creating any object**.

If `main()` were non-static, the JVM would first need to create an object of the class, which it doesn't do automatically.

---

## `void`

### Meaning

* Specifies that the method **does not return any value**.

### Why is it needed?

The JVM does not expect any value back after the program finishes execution.

---

## `main`

### Meaning

* Name of the method.

### Why is it needed?

The JVM looks specifically for a method named **`main`** as the starting point of the program.

Method Signature:

```java
public static void main(String[] args)
```

---

## `String[] args`

### Meaning

* An array of `String` objects.
* Stores **command-line arguments** passed while running the program.

Example:

```bash
java Main Hello Java
```

Then,

```java
args[0] = "Hello"
args[1] = "Java"
```

Example:

```java
public class Main {

    public static void main(String[] args) {

        System.out.println(args[0]);

    }
}
```

Output:

```text
Hello
```

# Summary Table

| Keyword         | Meaning                  | Why Required?                                    |
| --------------- | ------------------------ | ------------------------------------------------ |
| `public`        | Accessible from anywhere | JVM must access the method                       |
| `static`        | Belongs to the class     | JVM calls it without creating an object          |
| `void`          | No return value          | JVM doesn't expect any value back                |
| `main`          | Method name              | JVM starts execution from this method            |
| `String[] args` | Command-line arguments   | Receives values passed while running the program |
