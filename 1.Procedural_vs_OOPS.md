# Java Quick Notes

# 1. Procedural Programming vs Object-Oriented Programming (OOP)

| Procedural Programming                  | Object-Oriented Programming (OOP)                      |
| --------------------------------------- | ------------------------------------------------------ |
| Program is divided into **functions**.  | Program is divided into **classes and objects**.       |
| Focuses on **how** tasks are performed. | Focuses on **objects** and their interactions.         |
| Data and functions are separate.        | Data and methods are bundled together (Encapsulation). |
| Less reusable.                          | Highly reusable through Inheritance and Polymorphism.  |
| Less secure.                            | Better data security using access modifiers.           |
| Best for small applications.            | Best for large, scalable applications.                 |
| Example: **C**                          | Examples: **Java, C++, C#**                            |

### Example

**Procedural Programming (C)**

```c
int add(int a, int b) {
    return a + b;
}
```

**Object-Oriented Programming (Java)**

```java
class Calculator {
    int add(int a, int b) {
        return a + b;
    }
}
```

### Interview Points

* **Procedural Programming** → Function-oriented.
* **OOP** → Object-oriented.
* Java primarily follows the **Object-Oriented Programming** paradigm.
