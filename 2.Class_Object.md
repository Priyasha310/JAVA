
# 2. Class and Object

## Class

A **class** is a **blueprint/template** used to create objects.

It defines:

* Properties (State)
* Methods (Behavior)

### Example

```java
class Student {
    int rollNo;
    String name;
}
```

Think of a class as the **design of a house**.

---

## Object

An **object** is a real instance of a class.

Every object has:

### 1. State (Properties/Attributes)

Stores the data of an object.

Examples:

* rollNo
* name
* age

### 2. Behavior (Methods/Functions)

Defines what the object can do.

Examples:

* study()
* attendClass()
* displayDetails()

### Example

```java
class Student {
    int rollNo;
    String name;

    void study() {
        System.out.println(name + " is studying");
    }
}

public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();

        s1.rollNo = 101;
        s1.name = "Rahul";

        s1.study();
    }
}
```

### Output

```
Rahul is studying
```

## Key Difference

| Class                                     | Object                                |
| ----------------------------------------- | ------------------------------------- |
| Blueprint or template                     | Instance of a class                   |
| Logical entity                            | Physical entity                       |
| No memory allocated until object creation | Memory is allocated when created      |
| Used to define properties and methods     | Used to access properties and methods |

---

## Memory Trick

**Class = Blueprint 📝**

**Object = Real Thing 🏠**

**Object = State + Behavior**

* **State** → Variables (Properties)
* **Behavior** → Methods (Functions)

---

## Interview One-Liners

* **Class:** A blueprint or template from which objects are created.
* **Object:** An instance of a class that contains **state (properties)** and **behavior (methods)**.
* **State** is represented by variables.
* **Behavior** is represented by methods.
* Java is an **Object-Oriented Programming** language.
